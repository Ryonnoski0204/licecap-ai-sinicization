# 渐进式响应布局 + ≡ 控制按钮(Windows)— 设计文档

- 日期:2026-06-24
- 分支:`right-click-menu`(原分支 `main`)
- 目标平台:仅 Windows

## 1. 目标(第三版,渐进式)

窗口宽度变化时,底部按钮**渐进式逐个隐藏**(而非一刀切全隐藏);当第一个按钮(录制到剪贴板)放不下时,右下角出现 **≡** 控制按钮作为兜底入口;之后每变窄一点再隐藏一个按钮,最窄时只剩 ≡。被隐藏的功能都可通过 ≡ 的菜单访问(左键点击或右键 ≡)。

各宽度档(用户确认的方案 A):
```
够宽:  [帧率][尺寸][录到剪贴板]   [插入][录制][停止]      (≡ 隐藏)
渐窄:  [帧率][尺寸]   [插入][录制][停止] [≡]              (录到剪贴板放不下 → ≡ 出现)
更窄:  [录制][停止] [≡]                                   (尺寸→帧率→插入 依次隐藏)
最窄:  [≡]                                                (全部走 ≡ 菜单)
```

## 2. 关键设计

### 2.1 隐藏顺序由物理布局自然产生
LiceCap 底部布局:左组(帧率/尺寸/录制到剪贴板)**左锚定**固定;右组(插入/录制/停止)+ ≡ **右锚定**随窗口移动。窗口变窄时右组左移覆盖左组,碰撞顺序天然是:录制到剪贴板 → 尺寸 → 帧率(左组被覆盖)→ 插入 → 录制 → 停止(右组移出左界)。正好符合需求,无需自定义优先级。

### 2.2 ≡ 控制按钮的独立位置
≡ 占据最右下角的独立位置(右锚定),录制/停止/插入 相应左移让位:
```
插入 400 / 录制 443 / 停止 486 / ≡ 518  (单位 DLU)
```
正常宽度下 ≡ 隐藏(右下角有约一个按钮宽的空位,可接受)。

### 2.3 显隐判定(重写 UpdateDimBoxes)
以 ≡ 的左边为右边界基准 `leftmostRight`:
- **右组**(停止/录制 任意状态可见,插入仅暂停):按钮 `last.left < 4`(几乎移出左界)→ 隐藏;否则显示并更新 `leftmostRight = min(可见右组按钮.left)`。
- **左组**(仅停止状态显示):`last.right > leftmostRight - 4`(被右组覆盖)→ 隐藏。
- **状态栏**:录制中 且 未发生隐藏时显示。
- **≡**:只要发生任一隐藏(`anyHidden`)就显示。
- 保留原有"停止状态下更新尺寸输入框数值"逻辑。

### 2.4 最小宽度
`MIN_SIZE_X` 160 → **96**(像素),使窗口能窄到容纳 ≡ 而其余按钮全部隐藏(高 DPI 下可达"只剩 ≡")。

### 2.5 ≡ 菜单(沿用已实现)
左键 `WM_COMMAND(IDC_CTRL)` / 右键 `WM_CONTEXTMENU`(目标为 ≡)→ `ShowRecordContextMenu`(随状态:录制/录制到剪贴板/暂停/继续/停止/退出)。

## 3. 影响文件
- `licecap/resource.h`(IDC_CTRL,已存在)

## 4. 验证
- CI 编译通过;本机多档宽度截图:够宽(无 ≡、全按钮)→ 渐窄(录到剪贴板隐藏、≡ 出现)→ 更窄(逐个隐藏)→ 最窄(只剩 ≡);点击/右键 ≡ 弹菜单。
